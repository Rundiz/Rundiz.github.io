Class Languages

Work as connector between project's source code and oscarotero/Gettext class.

The translation text file (.mo) must store in the path like this Modules[module_name]\languages[text_domain]-[language_locale_uri].mo
The translation text file must have its name and follow with language_locale_uri which is refer from languages table in th database.

This class was initialized with I18n middleware class.

To use:

$Languages = new \System\Libraries\Languages('th');// change from "th" to your language locale URI (This is automatically initialize in the controller, just use $this->Languages instead).
$Languages->bindTextDomain('mytextdomain', 'Modules\MyModule\languages');// load translation text with bindTextDomain() method. The directory path should stay in the guideline above.
// call to getInstance() method to start using this class as static method such as gettext method.
$Languages->getInstance();
// translate the text.
echo \System\Libraries\Languages::gettext('Hello world', 'mytextdomain');

To use inside controllers:
Do not need to initialize the class and just use $this->Languages.

$this->Languages->bindTextDomain('mytextdomain', 'Modules\MyModule\languages');
$this->Languages->getInstance();
echo \System\Libraries\Languages::gettext('Hello world', 'mytextdomain');
