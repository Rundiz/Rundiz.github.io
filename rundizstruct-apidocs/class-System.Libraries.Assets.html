Class Assets

The assets class to work with add (enqueue), remove (dequeue), render the css, js files.

The asset locations are at public/assets/*, public/themes/[theme_system_name]/assets/* folders.

Namespace: System\Libraries
Author: Vee W.
Located at System/Libraries/Assets.php

# addAsset( string $type, string $handle, string $file, array $dependency = [], string|boolean $version = true, array $attributes = [], string|null $group = null )

Add an asset to the queue by type.

Add an asset to the queue by type.

Parameters

$type
The asset type (css, js).
$handle
The handle name of the asset. It must be unique to prevent overriding and duplication.
$file

The URL to file, it can be full URL or just relative path. To get relative path use \System\Libraries\Url() class and getPublicUrl() or getThemeUrl() methods. If the file has special characters, it should be urlencode() before calling this.

$dependency

An array of added asset handles that this asset is depend on. The css and js dependency cannot be cross type. Please try to use dependency for each group don't cross groups.

$version
The version number to added to the query string of this asset URL. If false means no version is added.
$attributes
An array of HTML attributes for this asset. Do not add id attribute because it will be auto generate.
$group
The group name. This can be helped when you want to render the assets by group.

Throws

InvalidArgumentException
Throw the errors if invalid argument is specified.
