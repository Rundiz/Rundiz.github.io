protected string
# locateTemplate( string $views_file )

Find the location of the template file.

Find the location of the template file.

This method was called by the render() method.

If it is admin pages then it will be searching the template file in these locations by order.
1. Modules/[current_module]/Theme/Admin/$views_file.
2. System/Core/Theme/Admin/Modules/[current_module]/$views_file.

If it is not admin pages, then it will be searching the template file in these locations by order.
1. public (or public_html folder where index.php is)/Themes/[theme_system_name]/Modules/[current_module]/$views_file.
2. Modules/[current_module]/Theme/Front/$views_file.
3. System/Core/Theme/Front/Modules/[current_module]/$views_file.

Parameters

$views_file
Path to views file. Refer from [theme_base_location]/Modules/[current_module]/your/path/to/views-file/here. Do not include the file extension.

Returns

string
Return the full path to the template file. If it is not found then return null.

# views( string $views_file, array $output_data = [] )

Load the views file manually.

Load the views file manually.

For load partial views file or based template which is out of locateTemplate() locations.
This method use less strict folder structure than render() method.

It will be searching the template file in these locations by order.
1. public (or public_html folder where index.php is)/Themes/[theme_system_name]/$views_file.
2. Modules/$views_file.
3. System/Core/Theme/$views_file.

Parameters

$views_file
Path to views file. Refer from [theme_base_location]/your/path/to/views-file/here. Do not include the file extension. For more information, please see locateViews() method.
$output_data
The data that will becomes variable in views file. Example ['foo' => 'bar']; will becomes echo $foo; // result is bar.

Returns

string
Return the views content. This method will not return ResponseInterface like render() did.
